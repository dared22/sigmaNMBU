# Site Overview: SIGMA — Data Science Branch

## 1. Core Mission

The Sigma Data Science Branch website serves as a high-fidelity portal for the student association, blending community engagement with rigorous technical aesthetics. It positions Sigma as a cutting-edge organization at the intersection of Data Science, AI, and Machine Learning.

---

## 2. Target Audience

| Audience | Purpose |
|----------|---------|
| **Students** | Looking for technical growth, events, and community |
| **Companies** | Seeking high-level talent and collaboration opportunities in DS/ML |
| **Faculty** | Interested in the bridge between academic theory and practical application |

---

## 3. Information Architecture

### A. Home *(Hjem)*
- **Hero** — "Welcome to Sigma // Data_Science_Branch" – A high-impact entry point with visual data representations
- **System Output** — A live stream of association updates (News) presented as terminal logs
- **Member Portal** — A technical login module for accessing datasets and community resources

### B. Events *(Arrangementer)*
- **Event_Log** — A searchable, filterable repository of upcoming workshops, lectures, and social "subroutines"
- **Technical Metadata** — Every event includes complexity levels `O(n)`, timestamps, and capacity metrics
- **Registration** — `Register_Node` actions for seamless signup

### C. About Us *(Om oss)*
- **Vision_Dict** — The core identity defined through a code-based mission statement
- **Training Log** — A chronological "epoch-based" history of the association's growth
- **Active Nodes** — Profiles of the Board of Directors, categorized by class: `Root`, `Daemon`, `Allocator`

### D. For Companies *(For bedrifter)*
- **B2B_Integration** — A professional portal for corporate partners
- **System_Metrics** — Data-driven proof of impact (active members, engagement rates)
- **Collaboration_Endpoints** — Modular sections for company presentations and technical workshops
- **Request_Protocol** — A structured intake form for new partnerships

---

## 4. Key UX Principles

| Principle | Description |
|-----------|-------------|
| **Modularity** | Information is contained within "windows" or "containers" that feel like an IDE |
| **Transparency** | Use of technical metadata to show, not just tell, the "Data Science" focus |
| **Efficiency** | Clear calls to action — `Execute`, `Register`, `Initialize` — aligned with the technical theme |
